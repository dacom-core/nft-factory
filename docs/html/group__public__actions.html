◆ acceptreq()

void nft::acceptreq ( eosio::name  seller,
uint64_t  request_id 
)

Метод принятия запроса на покупку

AUTH: seller

Продавец должен вызвать этот метод для начала сделки обмена, если:

  • установлен флаг доступности торгов по NFT
  • установлен флаг физической поставки (для ручного подтверждения адреса и введенных пользователем полей) Если флаги не установлены, вызов этого метода не требуется - обмен токен/NFT произойдёт автоматически.
Parameters
[in]sellerПродавец
[in]request_idИдентификатор запроса на покупку

Метод добавления пользователя в белый список для выпуска частей NFT.

AUTH: creator

Создатель NFT может сам выпускать части или добавить подрядчиков, которые поставляют части NFT в его объект. Для этого, создатель добавляет подрядчика в белый лист с помощью этого метода.

Parameters
[in]creatorВладелец NFT
[in]object_idИдентификатор объекта
[in]usernameИмя пользователя для добавления в белый лист

uint64_t  request_id,
eosio::name  status

Метод установки статуса доставки

AUTH: delivery_operator

После того, как сделка на покупку NFT с доставкой оплачена, подключается оператор доставки, который изменяет статус доставки. Сейчас доступен только статус finish, что означает, что объект выдан покупателю на пункте самовывоза. Оператором доставки, при этом, выступает продавец.

Parameters
[in]delivery_operatorОператор доставки
[in]request_idИдентификатор запроса на покупку
[in]statusСтатус доставки
